执行junit单元测试出错
@ 嚣张协奏曲 | 星期日，十二月 27 日，2020 年 | 1 分钟阅读 | 更新于 星期日，十二月 27 日，2020 年

通过IDEA执行的junit单元测试时，出现程序包不存的问题，提示"java: 程序包org.springframework.stereotype不存在"

环境

  • Java 8
  • Maven 3.5
  • Junit 1.2
  • IDEA 2020.1

回顾

  1. 正常创建maven
  2. 在项目右键选择Add Framework support
  3. 选择Spring框架支持，IDE自动生成配置文件
  4. 在pom文件编写依赖
  5. 正常编写业务逻辑
  6. 点击IDEA提供的测试按钮

现象

  • 通过IDEA执行的junit单元测试时，提示程序包不存
  • 但通过maven查看包依赖并无异常
  • 使用maven的test命令，发现成功执行，无异常

分析

IDEA在执行编译测试时并没有使用maven解决依赖问题

解决

让IDEA在构建时通过maven

  1. 打开Setting
  2. 搜索Runner
  3. 勾选Delegate IDE build/run actions to Maven

java maven IDEA
