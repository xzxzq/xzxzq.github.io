HUGO-静态博客的利器
@ 嚣张协奏曲 | 星期五，十二月 25 日，2020 年 | 5 分钟阅读 | 更新于 星期五，十二月 25 日，2020 年

为了这个他世界操碎了心，终于那个人准备重出江湖搭建了博客…

1、基础环境

  • win10操作系统

    本人所有操作都在windows系统下进行

  • gti环境

    同步博客的重要工具，使用方式本文不介绍

  • Gitee或GitHub账号

    静态博客的搭建依托与这两个平台提供的page服务

2、装配HUGO

  • 下载本人创建这个文章的时候的版本：hugo 0.79.1 win64版本下载

    也可直接前往官方仓库hugo下载，选择基础版本

  • 下载好的压缩包里有仅一个可执行文件，将他解压到软件的安装目录例如D:\Software\hugo

  • 将hugo加入环境变量

    • win + r打开运行，输入sysdm.cpl，打开系统属性页面，选择"高级"，点击"环境变量"
    • 在path里面加入hugo.exe的当前路径，即D:\Software\hugo

  • 测试一下hugo能不能用，在命令行中输入hugo version，出现如下字样就是设置好了

    ❯ hugo version
Hugo Static Site Generator v0.79.1-EDB9248D windows/amd64 BuildDate: 2020-12-19T15:44:19Z

3、创建hugo站点

站点也就是hugo的工作目录，生成静态页面的所需要的环境。

打开命令行，进入到一个合适的目录输入命令

❯ hugo new site [指定目录]

执行后就会在指定的目录生成基本文件，当出现如下结果就是成功创建

❯ hugo new site sample
Congratulations! Your new Hugo site is created in D:\WorkSpase\study\test\sample.

Just a few more steps and you're ready to go:

1. Download a theme into the same-named folder.
   Choose a theme from https://themes.gohugo.io/ or
   create your own with the "hugo new theme <THEMENAME>" command.
2. Perhaps you want to add some content. You can add single files
   with "hugo new <SECTIONNAME>\<FILENAME>.<FORMAT>".
3. Start the built-in live server via "hugo server".

Visit https://gohugo.io/ for quickstart guide and full documentation.

4、安装一个主题

在官网的主题商城上选择一个主题，下载并解压到站点themes目录下，例如我选择Stack，解压的根目录一定得是"hugo-theme-stack"，否则hugo会找不到主题

或者通过git命令的方式下载

❯ cd .\themes\
❯ git clone https://github.com/CaiJimmy/hugo-theme-stack.git hugo-theme-stack

打开主题目录，复制archetypes到站点根目录，这里面是主题配套的文章模板文件

然后打开exampleSite，复制config.yaml到站点根目录，并删掉原来的config.toml文件

5、创建文章

就在当前目录下输入命令就会按照默认模板生成文件

hugo new post/test.md

这里test.md是我们需要创建的文件名字，一般使用markdown写博客，所以以需要.md结尾。如果创建成功，会在命令行输出生成的文件位置，如：

❯ hugo new test.md
D:\WorkSpase\study\test\sample\content\post\test.md created

现在就可以打开test.md进行编写，如：

❯ vim D:\WorkSpase\study\test\sample\content\test.md

---
title: "文章标题"
date: 2020-12-26T16:03:58+08:00
draft: true
categories:
    - 博客
tags:
    - Hugo
    - Stack
---
这里是主体内容
hello world

其中---所包围的部分是当前文章的一些配置

  • title：文章标题
  • date：创建日期
  • draft：草稿标记，设置为true时，hugo就会忽略这个文件
  • categories：主题的设置，用来设置文章分类
  • tags：主题的设置，用来设置文章标签

都写好后，需要把这个darft:true删除掉

6、编译和调试

直接在站点根目录命令行输入hugo就可以编译生成静态页面，同时会在根目录创建public文件夹

想要在本地调试，可以在命令行输入

❯ hugo server

7、创建远程仓库

7.1、Gitee

  • 先创建一个公开的仓库

  • 仓库名称与账户名一致（为了缩短url路径）

  • 在本地站点的public目录下初始化git仓库

    ❯ cd .\public\
❯ git init

  • 绑定远程仓库

    在仓库的主页复制git链接，其中git@部分要填写自己的仓库地址

    ❯ git remote add origin git@gitee.com:xzxzq/xzxzq.git

  • 打开gitee，找到刚创建的仓库主页，点page服务

  • 按照默认设置，点部署

    成功后会在页面当前页面显示已开启Gitee Pages服务，复制后面的网址地址

  • 修改站点根目录下的config.yaml文件，修改以下的属性

    baseurl: https://xzxzq.gitee.io
languageCode: zh-CN
title: XXX的博客

    baseurl填写刚才复制的地址，title网站标签页的标题，注意分号后面要有一个空格

  • 添加、提交、推送一气呵成

    ❯ git add *
❯ git commit -m "update config"
❯ git push --force origin master

  • 回到启用pages的页面，一定要再点击更新，这样再访问自己的页面时就会和本地调试时一致

其他问题可参考码云pages

7.2、 Github

  • 创建一个公开的仓库

  • 仓库的名字与github主页路径名一致（为了缩短url路径）

    这里提示错误信息是因为我已经创过了这个仓库

  • 同样绑定远程仓库，在仓库的主页复制git链接

    ❯ git remote add origin git@github.com:xzxzq/xzxzq.github.io.git

  • 打开github的仓库设置

    在仓库主页的上方有个Settings标签，点击进入

  • 选择最下面的Github Pages服务开启

    Branch选择master其他的默认就行，然后点Save，之后在上方会出现主页地址，复制它

  • 修改站点根目录下的config.yaml文件，修改以下的属性

    baseurl: https://xzxzq.github.io/

    baseurl填写刚才复制的地址，title网站标签页的标题，注意分号后面要有一个空格
